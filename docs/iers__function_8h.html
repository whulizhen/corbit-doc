◆ admint()

int admint ( const double *  ampin,
const int  idtin[][6],
const double *  phin,
double *  amp,
double *  f,
double *  p,
const int  nin,
int *  nout,
const int *  itm 
)

This function returns the ocean loading displacement amplitude, frequency, and phase of a set of tidal constituents generated by the Bos-Scherneck website at http://holt.oso.chalmers.se/loading/. The variable nin is input as the number wanted, and the variable nout is returned as the number provided. The constituents used are stored in the arrays idd (Doodson number) and tamp (Cartwright-Edden amplitude). The actual amp and phase of each of these are determined by spline interpolation of the real and imaginary part of the admittance, as specified at a subset of the constituents.

参数
[in]ampinCartwright-Edden amplitude of tidal constituents (size nin).
[in]idtinDoodson number of tidal constituents (size 6xnin).
[in]phinPhase of tidal constituents
[in]ninNumber of harmonics used
[in]itmDate as integer array in UTC. The format should be: [year,day_of_year,hours,minutes,seconds].
[out]ampAmplitude due to ocean loading
[out]fFrequency due to ocean loading
[out]pPhase due to ocean loading
[out]noutNumber of harmonics returned
返回
Always zero
注解
  1. The phase is determined for a time set in COMMON block /date/ in the function TDFRPH.
  2. The arrays F and P must be specified as double precision.
  3. Status: Class 1 model
版本
2009 August 19

◆ etutc()

double etutc ( const double  year)

The purpose of the function is to compute the difference, delta, between Epheremis Time (ET) and Coordinated Universal Time (UTC).

参数
[in]yearDecimal year (Note 1)
返回
Delta ET - UTC (Note 2)
注解
  1. This function is valid only from 1700.-until next leap second. Currently, this is up to 2012.5.
  2. The expression used in given in seconds.
  3. Leap second table in GAMIT UTC (and UT) is the time most often used (e.g. in time signals)
  4. Status: Canonical model
警告
IMPORTANT
A new version of this routine must be produced whenever a new leap second is announced. There are three items to change on each such occasion:
1) Update the nstep variable
2) Update the arrays st and si
3) Change date of latest leap second
Latest leap second: 2012 June 30

◆ eval()

int eval ( const double  y,
const int  nn,
const double *  x,
const double *  u,
const double *  s,
double *  val 
)

This function performs cubic spline interpolation of a given function sampled at unequally spaced intervals. The function spline needs to be called beforehand to set up the array s.

参数
[in]yThe coordinate at which a function value is desired (Note 1)
[in]nnNumber of samples of the original function
[in]xArray containing sample coordinates x(1),x(2),...x(nn) (Note 2)
[in]sArray containing the 2nd derivatives at the sample points (Note 3)
[in]uArray containing samples of a function at the coordinates x(1),x(2),...x(nn)
[out]valThe interpolated value of the function at y
返回
Zero on success; else 1.
注解
  1. If y falls outside the range (x(1),x(nn)), the value at the nearest endpoint of the series is used.
  2. The sequence x(1),x(2),...x(nn) must be strictly increasing.
  3. This array is found by the function SPLINE, which must be called once before beginning this interpolation.
版本
2009 August 19

◆ shells()

int shells ( double *  x,

This function sorts an array x, of length n, sorting upward, and returns an array k which may be used to key another array to the sorted pattern (i.e., if we had an array f to which x corresponded before sorting, then after calling SHELLS, f(k(1)) will be the element of f corresponding to the smallest x, f(k(2)) the next smallest, and so on).

参数
[in]xArray to be sorted (Note 1)
[in]nLength of input array x.
[out]kSorted array that may be used to key another array.
返回
Zero on sucess, 1 for failure.
注解
  1. See the subroutine ADMINT header comments for detailed information.
  2. The original fortran routine used a shell sorting algorithm (hence the name). This function does tha same as the original, but via a completely different way.
  3. Status: Canonical model

◆ spline()

int spline ( const int  nn,
const double *  x,
const double *  u,
double *  s,
double *  a 
)

The purpose of this function is to find an array s for the spline interpolator function EVAL.

参数
[in]nnNumber of data points supplied, which may be negative (Note 1)
[in]xArray containing x-coordinates where function is sampled (Note 2)
[in]uArray containing sample values that are to be interpolated
[in]aWorking space array of dimension at least nn
[out]sOutput array of 2nd derivative at sample points
返回
Always zero
注解
  1. If the user wishes to force the derivatives at the ends of the series to assume specified values, he or she should put du(1)/dx and du(n)/dx in the variables s1 and s2 and call the subroutine with nn = -(number of terms in the series). Normally a parabola is fitted through the 1st and last 3 points to find the slopes. If less than 4 points are supplied, straight lines are fitted.
  2. The sequence xx(1), xx(2), ... xx(nn) must be strictly increasing.
  3. Status: Canonical model

◆ tdfrph()

int tdfrph ( const int *  idood,
const int *  itm,
double *  freq,
double *  phase 
)

This subroutine returns the frequency and phase of a tidal constituent when its Doodson number is given as input.

参数
[in]idoodDoodson number of a tidal constituent
[in]itmDate as integer array in UTC. The format should be: [year,day_of_year,hours,minutes,seconds].
[out]freqFrequency of a tidal constituent
[out]phasePhase of a tidal constituent (Note 1)
注解
  1. The phases must be decreased by 90 degrees if the sum of the order and the species number is odd (as for the 2nd degree diurnals, and 3rd degree low frequency and semidiurnals). These phases may need further adjustment to allow for the spherical harmonic normalization used; e.g. for that used for the potential by Cartwright and Tayler, 180 degrees must be added for (species, order) = (1,2), (1,3), or (3,3).
  2. Status: Class 1 model
